📋 Anganwadi Survey System — How To Guide

Survey Link (never changes): https://dgmahajan.github.io/Anganwadi-Survey

Google Sheet: Anganwadi Survey (in your Google Drive)

Key idea: For each new survey, you give your developer the information below. He creates one file and sends it to you. You upload it to GitHub. That's all.

1. Starting a New Survey

Give your developer the following 7 things:

#What to giveExample
1Survey name (in Marathi)ऑफ-ग्रिड सौर ऊर्जा योजना स्थिती सर्वेक्षण
2Subtitle (in English)Murbad Taluka — off-grid solar power scheme Status Survey
3Start date2026-04-28
4End date2026-05-31
5Sheet tab name (what to call the tab in Google Sheet)Murbad Anganwadi 2026
6Anganwadi list (ID, name, taluka) — paste from ExcelSee below
7Questions — write them out in plain languageSee below

Question types available:

  • Yes / No — simple two-button choice
  • Yes / No / Don't Know — three-button choice
  • Number — worker types a number (you specify min and max)
  • Text — worker types free text (can be optional)
  • Any question can be set to appear only if a previous question was answered in a specific way

2. What Your Developer Gives You Back

He will give you two files:

  • surveys/index.js — updated master list
  • surveys/[survey-name].js — the new survey file

3. Uploading to GitHub

  1. Go to github.com/dgmahajan/Anganwadi-Survey
  2. Click Add file → Upload files
  3. Drag both files into the upload area
  4. Click Commit changes
  5. Wait 1–2 minutes — the survey goes live automatically
⚠️ When uploading the survey file, make sure the filename includes the folder name.
For example: the file should be inside a folder called surveys on GitHub.
Your developer will tell you exactly which files to upload each time.

4. Sharing the Survey

  • Send this single link to the Taluka administrator: https://dgmahajan.github.io/Anganwadi-Survey
  • He shares it in his WhatsApp group
  • Anganwadi workers open it on their phone and fill the form
  • The link never changes — same link for every survey

5. Checking Responses

  1. Go to sheets.google.com
  2. Open Anganwadi Survey
  3. Each survey has its own tab at the bottom — click on it to see responses

6. How the Survey Closes

  • The survey closes automatically on the end date you gave
  • After that date, workers see "No active surveys" — they cannot answer
  • Old responses remain safe in the Google Sheet
  • If you want to close early — tell your developer, he changes the end date and you upload

7. If a Worker Makes a Mistake

  • Worker opens the link again, selects their Anganwadi
  • Their previous answers appear on screen automatically
  • They correct what is wrong and resubmit
  • The old answer is replaced with the new one in Google Sheet

8. Administrator Preview (TEST entry)

  • Every survey has a special entry: ID = TEST, Name = Test Anganwadi (Admin Preview)
  • The Taluka administrator selects these to preview and test the survey
  • TEST submissions are saved to the sheet like any other Anganwadi

9. Checking Which Anganwadis Have NOT Yet Responded

Open this link in your browser:

https://dgmahajan.github.io/Anganwadi-Survey/pending.html

You will see:

  • Total Anganwadis / Submitted / Pending count for each active survey
  • A table of all Anganwadis that have not yet submitted — with ID and name
  • Download CSV — saves the list to your computer (opens in Excel)
  • Copy List — copies ID + Anganwadi name to clipboard (paste into WhatsApp or email)
⚠️ This page requires the Apps Script to be updated once by your developer.
He adds a small code block (apps-script-pending-action.js) to the Apps Script and re-deploys it. After that, the pending page works automatically for all future surveys too.

10. Adding More Anganwadis After Survey is Live

  • Tell your developer — give him the new IDs and Anganwadi names
  • He updates the survey file and gives it back to you
  • You upload to GitHub — new Anganwadis appear in the dropdown immediately
  • Anganwadis that already submitted are not affected
